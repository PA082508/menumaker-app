## §4 Родительская часть — по факту кода (честно)

- **Живо:** вход по телефону + OTP, выбор ребёнка, Drop Off / Pick Up (прямой INSERT в `safepass_sessions`, не RPC), кнопка Remind, экран `waiting` → `confirmed` по realtime.
- **Журнала нет.** На `home` рисуется **одна** последняя запись за сегодня (`todaySessions[0]`), остальное — мёртвое состояние. Подписки на `home` нет: если подтвердить, пока родитель сидит на `home`, экран не обновится до перехода.
- **OTP** генерируется **в браузере самого родителя** и печатается в console — доставки (SMS/email) не существует. Поэтому тест идёт через тестовый номер.
- **Redeem-клиент — НЕТ.** `safepass_redeem_temp_code` есть в базе и выдан `anon`, но вызовов в коде **ноль** (ни кнопки, ни QR). И он в принципе кассирский (нужен device token + PIN), не родительский.

## §6 Родительская запись — РЕШЕНО, ждёт применения (файл `20260719b_safepass_parent_rpc.sql`)

`anon` не имеет INSERT на `safepass_sessions`, поэтому клиентская правка `classroom_id`
ничего не спасает. Решение одним движением — anon-вызываемый `SECURITY DEFINER` RPC
`safepass_request_handoff(p_phone, p_child_id, p_action)`, который сам:
проверяет, что телефон есть в `safepass_trusted_persons` для этого ребёнка;
**берёт `classroom_id` из `roster` по `child_id`** (DECISIONS:210 — сервером, где права есть);
вставляет строку. Это новая миграция → отдельный go.
