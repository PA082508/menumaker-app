# Ворота деплоя вместо имени ветки — предложение

**Повод:** `bc07e18` с заголовком **«[BRANCH — do not deploy]»** уехал в main и уронил кухню.
Заказ: «предложи реальный деплой-гард вместо имени ветки — вариант + цена».

**Замер 2026-07-16:** `.github/workflows` — **не существует**. CI нет вообще. Vercel катит
`main` автоматически. Есть `npm run build` (`tsc && vite build`) и `npm test` (`vitest run`).
`gh` авторизован как `PA082508`.

---

## 0. Сначала — что бы НЕ помогло

Легко предложить «branch protection + CI» и закрыть вопрос. Но проверим честно:

**Авария кухни прошла бы через build и test.** Она была не ошибкой типов и не падением
теста: `v_meal_grid` не имеет `photo_url`, PostgREST отклонял select **в рантайме**, а
`tsc` про схему БД не знает ничего. **CI из build+test поймал бы ноль из двух вчерашних
аварий** (кухня и 360 семей — одна и та же ошибка).

Значит «включить CI» — не ответ на этот вопрос. Это отдельная польза.

## Вариант 1 — ⭐ Гард под НАШУ поломку: select vs каталог

**Что:** тест, который парсит все `.from('X').select('a,b,c')` в `src/` и сверяет каждую
колонку с `information_schema.columns` для этой реляции. Падает — билд красный.

**Почему именно он:** он поймал бы **обе** вчерашние аварии — и `guardian.relationship`
(360 семей), и `v_meal_grid.photo_url` (кухня). Это единственный предложенный гард,
который ловит уже случившееся, а не воображаемое.

**Цена:**
- ~100–150 строк: regex по `.from(...)` + `.select(...)`, запрос к `information_schema`, диф.
- нужен доступ к БД из CI → `SUPABASE_DB_URL` в secrets (read-only роль достаточно);
- ложные срабатывания: динамические select'ы (`select('*')`, шаблонные строки) — пропускать, не гадать;
- ~1 заход.

**Ограничение, честно:** он не увидит select, собранный из переменной. Ловит подавляющее
большинство — те, что написаны литералом, как обе вчерашние.

## Вариант 2 — маркер-гард

**Что:** CI-шаг, падающий, если тема любого коммита в PR матчит
`/do not (deploy|merge)|DO NOT|WIP|HOLD/i`.

**Почему:** превращает намерение из сообщения в **проверяемое условие**. Ровно та дыра,
через которую прошёл `bc07e18`.

**Цена:** ~10 строк YAML. Бесплатно.
**Ограничение:** работает **только вместе с вариантом 3** — без защиты ветки красный CI
на main это констатация постфактум, а не ворота.

## Вариант 3 — branch protection на `main`

**Что:** GitHub: `main` требует PR + зелёные проверки. Прямой push запрещён.

**Цена — и она настоящая:** сегодня в main **25 прямых пушей**. Аварию кухни я чинил
пушем в main через ~20 минут после обнаружения. С защитой это PR + ожидание CI (~2–3 мин)
+ merge. Для аварии кухни это приемлемо; для «поправить опечатку в спеке» — трение.

**Смягчения:**
- админ может обходить (`Do not allow bypassing` — **не включать**), то есть в аварии путь остаётся;
- `gh pr create --fill && gh pr merge --auto --squash` — одна команда, а не ритуал;
- защита только от **чужого** случайного мержа — но чужих здесь нет, репозиторий ваш.

**Мой честный вывод по 3:** это единственное, что делает 2 воротами, но цена — темп.
При одном человеке в репозитории защита ветки ловит в основном **себя же**. Стоит того,
только если 2 включён — иначе это ритуал без улова.

## Вариант 4 — Vercel Ignored Build Step

**Что:** Vercel умеет «Ignored Build Step» — команда, чей exit 0 пропускает билд.
Можно проверять сообщение коммита.

**Цена:** ~5 строк. **Но:** останавливает только фронт. Вчерашняя авария пришла бы всё
равно — код-то был правильный, ломала **схема**. И миграции Vercel не касается вообще.
**Не рекомендую как основной гард.**

ехали одной веткой. Ворота бы это поймали; разделение — предотвратило бы.
